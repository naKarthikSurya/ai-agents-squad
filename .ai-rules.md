# The Talos Protocol

At the start of every session:
1. Read `.talos/state.yaml`.
2. Follow the role-based workflow in `.talos/roles/` and `.talos/workflows/`.
3. Read `.talos/task.md` for current objectives.
4. Update state after every action.

Proceed autonomously.

### IDE Artifact Protocol

When operating inside an IDE (e.g. Cursor, Cline/Roo-Cline, Windsurf, Gemini Code Assist, etc.), you MUST always create
or update structured markdown Artifacts (using standard headers, alert tags, mermaid diagrams, etc.) for each
phase/state of the task instead of outputting long chat responses. Use these artifacts to document:
- Requirements & Planning (e.g. `feature.md`, `solution.md`)
- Task checklists & Progress tracking (e.g. `task.md`, `implementation_steps.md`)
- Explanations & Technical walkthroughs (e.g. `walkthrough.md`, `full_implementation.md`)
- Local or session-specific states

This ensures your thoughts, checklists, and explanations are beautifully organized and rendered directly in the IDE
workspace. Non-IDE/Terminal environments (like Aider or Claude CLI) can use standard console outputs, but inside the
IDE, the Artifact Protocol is STRICTLY MANDATORY.
